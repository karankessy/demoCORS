🌐 CORS Policy Demo

Frontend on localhost:3000 → API on localhost:4000

How it works:
This page is served from http://localhost:3000. It makes fetch() requests to an API on http://localhost:4000.
Since these are different origins (different ports), the browser enforces CORS (Cross-Origin Resource Sharing) policy.

🔴 Without CORS headers — the browser blocks the response.
🟢 With CORS headers — the server explicitly allows the frontend origin, so the request succeeds.

🔴 Without CORS Blocked

Backend: server-no-cors.js (no CORS headers)

Click a button to make a cross-origin request... The browser will BLOCK the response because the server does not send the required Access-Control-Allow-Origin header.

🟢 With CORS Allowed

Backend: server-with-cors.js (CORS enabled)

Click a button to make a cross-origin request... The server includes the header: Access-Control-Allow-Origin: http://localhost:3000 so the browser allows the response.
