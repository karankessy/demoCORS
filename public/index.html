🌐 CORS Policy Demo

Frontend on localhost:3000 → API on localhost:4000

How it works:
This page is served from http://localhost:3000. It makes fetch() requests to an API on http://localhost:4000.
Since these are different origins (different ports = different origin), the browser enforces CORS (Cross-Origin Resource Sharing) policy.

🔵 Same-Origin — requests to localhost:3000 always succeed (no CORS involved).
🔴 Without CORS headers — the browser blocks the cross-origin response.
🟢 With CORS headers — the server explicitly allows the frontend origin.
🟡 Preflight — the browser sends an OPTIONS request first for non-simple requests.

② Preflight Requests (POST with JSON + Custom Headers)

🔴 Preflight Blocked No CORS

POST with Content-Type: application/json triggers OPTIONS preflight

Click to trigger a preflight request. POST with application/json is NOT a "simple request" so the browser sends OPTIONS first. Without CORS headers, preflight fails and the actual POST is never sent.
